# 🚀 HR AI 서류 진위확인 솔루션 — 로컬 구동 및 설정 가이드 (RUN_GUIDE.md)

이 문서는 로컬 개발 환경에서 Next.js 웹 애플리케이션 및 RPA 검증 시스템을 구동하기 위한 가이드입니다. 
프로그램을 실행하고 테스트하는 과정에서 발생할 수 있는 문제와 이에 대한 해결법을 기록하여, 추후에도 언제든지 참고하여 구동할 수 있도록 작성되었습니다.

## 📌 1. 필수 사전 요구사항 (Prerequisites)

이 솔루션은 백그라운드에서 **Node.js** 웹 서버가 동작하여 구동되므로 아래 프로그램들이 PC에 먼저 깔려 있어야 정상 작동합니다.

1. **Node.js 설치 (v18 이상 권장)**
   * Next.js 웹 서버 구동을 위해 반드시 필요합니다.
   * [Node.js 공식 홈페이지(https://nodejs.org)](https://nodejs.org)에서 **LTS(안정) 버전**을 다운로드하여 설치해 주세요.
2. **인터넷 연결**
   * 최초 실행 시 라이브러리 설치(`npm install`) 및 구글 Gemini API 통신을 위해 외부 인터넷망 연결이 필요합니다.

## 🔑 2. 환경 변수 및 API 키 설정 (`.env.local`)

API 키와 데이터베이스 경로 등 보안과 설정을 관리하기 위한 파일입니다. 이 설정이 올바르지 않으면 AI 판독이나 암호화 기능이 오동작합니다.

### ⚙️ 설정 방법
1. [nextapp/.env.example](file:///c:/Users/user/Desktop/vibe%20codeing/HR-booleanAI-project/HR%20BooleanAI-app/nextapp/.env.example) 파일을 복사합니다.
2. 복사한 파일 이름을 **`.env.local`** 로 변경하여 [nextapp/.env.local](file:///c:/Users/user/Desktop/vibe%20codeing/HR-booleanAI-project/HR%20BooleanAI-app/nextapp/.env.local) 경로에 둡니다.
3. 메모장이나 에디터로 열어 아래 주요 변수들의 값을 설정합니다.

### 📝 주요 변수 설명
* **`GEMINI_API_KEY`**: 구글 AI 스튜디오에서 발급받은 Gemini API 키 (서류 이미지 진위 분석 시 사용).
* **`ENCRYPTION_KEY`**: 기관 패스워드 암호화에 쓸 **64자리의 16진수 임의의 값** (예: `0123456789abcdef...`).
* **`SESSION_SECRET`**: 세션 암호화용 32자 이상의 난수 텍스트.
* **`RESEND_API_KEY`**: 이메일 알림 전송을 위한 Resend 서비스 키 (3단계 메일 알림 구현 시 사용).

> [!WARNING]
> `.env.local` 파일은 API 키와 같은 보안 정보가 들어가므로 **절대 Git에 올리면 안 됩니다.** (이미 `.gitignore` 규칙으로 커밋 대상에서 자동 제외 처리되어 있습니다.)

루트 디렉터리에 위치한 [불리언 진위확인 실행기.exe](file:///c:/Users/user/Desktop/vibe%20codeing/HR-booleanAI-project/%EB%B6%88%EB%A6%AC%EC%96%B8%20%EC%A7%84%EC%9C%84%ED%99%95%EC%9D%B8%20%EC%8B%A4%ED%96%89%EA%B8%B0.exe)를 이용해 서비스를 제어할 수 있습니다.

### 🔘 버튼 기능 설명
* **웹 대시보드 서버 시작**:
  * 백그라운드에서 Next.js 개발 서버(`npm run dev`)를 가동합니다.

* **서버 중지**:
  * 구동 중인 백그라운드 Node.js 서버 프로세스 트리를 강제 종료하여 포트 점유를 해제합니다.

* **DB 설정 및 동기화**:

## 🛠️ 4. 자주 묻는 질문 및 문제 해결 (Troubleshooting)

### Q. 실행기를 실행했는데 아무 반응이 없거나 서버 시작 단계에서 멈춥니다.
* **원인**: 시스템 환경변수(PATH)에 `node`나 `npm` 경로가 정상적으로 지정되지 않았거나 Node.js가 깔려 있지 않은 상태입니다.
* **해결**: [1단계]를 참고하여 Node.js를 정상 설치한 뒤 실행기를 완전히 껐다가 다시 구동해 주세요.

### Q. 'Port 3000 is already in use' (3000번 포트 충돌) 에러가 뜹니다.
* **원인**: 이전 서버 구동 프로세스가 비정상 종료되었거나, 다른 프로그램이 3000번 포트를 선점하고 있습니다.
* **해결**: 실행기에서 **[서버 중지]** 버튼을 다시 눌러 강제 킬(Kill)을 시도하거나, 명령 프롬프트(cmd)에서 `taskkill /f /im node.exe`를 치고 재구동해 주세요.

### Q. 웹 서버 접속 시 로그인하라고 나오는데 아이디가 무엇인가요?
* **원인**: MVP 단계로 현재는 DB에 기본 가입된 하드코딩 계정으로 접근할 수 있도록 설계되어 있습니다.

  * **일반 작업자 (Operator)**: `operator@hr-booleanai.com` / `Operator1234!`

## 📋 5. 기능 개발 진척 현황 (현재 개발 완료 vs 미완성)

현재 전체 로직의 대부분이 안정적으로 마이그레이션 되었으며 아래 2가지 기능의 비즈니스 연결을 남겨두고 있습니다.

* **완료된 기능 (100% 검증)**:
  * 웹 UI 구조 및 로그인 세션 제어 (/login, /dashboard, /settings/agencies)
  * SQLite DB 데이터 모델 설계 및 Prisma 구축
  * AES-256 패스워드 비밀키 보안 암호화 유틸리티
  * Puppeteer Stealth 브라우저 및 4단계 폴백 캡처 로직
  * Gemini AI 멀티모달 리뷰어 에이전트 연동
  * 자격/학적 유효기간 판정 및 4대보험 재직 경력사항 1:N 대조 알고리즘
* **미구현된 기능 (구조 설계 단계)**:
  * 수동 처리(에듀퓨어/윈스팩) 리스트 및 업로드 파일 엑셀 파싱 비즈니스 로직 연동
  * 최종 REJECT 결과물에 대한 인사담당자 이메일 자동 발송 연동 (Resend API)
