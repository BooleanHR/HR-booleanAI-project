# HR AI BooleanAI_Work Base - Wiki Schema

본 프로젝트는 `llm-wiki.ko.md`에서 제시하는 지식 베이스 아키텍처 패턴을 따릅니다. LLM 에이전트(작업자)는 이 스키마를 준수하여 `wiki` 디렉토리를 문서화, 유지보수, 종합해야 합니다.

## 1. 디렉토리 구조
- `raw/`: 원본 소스가 저장되는 곳입니다. (LLM은 이 폴더의 파일을 절대 수정하지 않고, 읽기만 수행합니다)
- `wiki/`: LLM이 생성하고 유지보수하는 마크다운 지식 베이스입니다.
  - `wiki/index.md`: 위키 전체 목록의 카탈로그. 페이지 추가 시 반드시 업데이트합니다.
  - `wiki/log.md`: 수집, 질의, 수정, 점검 등에 대한 연대기적 로그입니다.
  - `wiki/concepts/`: 시스템, 아키텍처, 비즈니스 로직 등 추상적인 개념 페이지가 저장됩니다.
  - `wiki/entities/`: 인물, 조직, 구체적인 컴포넌트 등의 엔티티 페이지가 저장됩니다.
  - `wiki/sources/`: `raw/`에 있는 원본 소스에 대한 메타데이터, 요약, 핵심 추출 내용이 저장되는 페이지입니다.

## 2. LLM 에이전트 작업 파이프라인
사용자의 요청에 따라 다음 워크플로우를 수행합니다.

### A. 수집 (Ingest) 파이프라인
1. `raw/` 디렉토리에 추가된 새 소스를 읽습니다.
2. `wiki/sources/` 에 새 하위 문서를 만들어 요약, 메타데이터, 주요 컨텍스트를 작성합니다.
3. 소스가 연관된 `wiki/concepts/` 또는 `wiki/entities/` 문서를 생성하거나 갱신하여 교차 참조(Cross-reference)를 연결합니다.
4. `wiki/index.md` 에 새 문서 정보 및 수정된 문서의 인덱스를 업데이트합니다.
5. `wiki/log.md` 에 `## [YYYY-MM-DD] ingest | 원본 파일명` 형태로 기록합니다.

### B. 질의 (Query) 및 종합 파이프라인
1. 질의 시, `wiki/index.md`를 먼저 참고해 위키에 축적된 지식을 파악한 후 문서를 찾습니다.
2. 여러 위키 페이지를 조합해 통찰(Insight), 비교 표, 분석 보고서 등을 도출합니다.
3. 도출된 분석 결과가 가치가 있다면(예: 새로운 인사이트), `wiki/concepts/` 혹은 최상위 `wiki/`에 새 문서로 종합(Synthesize)하고 `wiki/index.md`에 추가합니다.
4. `wiki/log.md`에 `## [YYYY-MM-DD] query/synthesis | 분석 내용` 형태로 로그를 남깁니다.

### C. 점검 (Lint) 파이프라인
1. 위키 페이지 간의 고립된 문서(Orphan), 깨진 링크, 상충하는 사실, 구조적 결함을 찾아냅니다.
2. 내용을 일치시키고 양방향 링크를 강화하며 누락된 정의를 채워넣습니다.
3. `wiki/log.md`에 결과를 로깅합니다.

## 3. 원칙사항
- **위키는 LLM이 전적으로 관리합니다.** 사용자는 읽거나 방향을 지시할 뿐이며, 문서화의 실질적 작성과 구조 유지는 LLM에 의해 이루어집니다.
- **모든 문서는 상호 참조를 가져야 합니다.** 고립되지 않도록 문서 내 연관 링크를 마크다운 형태(`[문서명](경로)`)로 유지하세요.
- 언제나 변경 사항을 발생시킨 경우, **index.md와 log.md의 업데이트를 잊지 마십시오.**
