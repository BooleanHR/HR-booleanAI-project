# HR BooleanAI — AI 기반 HR 서류 진위확인 시스템 UI 프로토타입

## 앱 개요

**앱 이름:** HR BooleanAI  
**목적:** 기업 HR 담당자가 지원자 제출 서류(졸업증명서, 자격증, 경력증명서 등)를 AI(Gemini Vision OCR)로 자동 진위확인하고, 결과를 검토·승인·반려하고, 불일치 발생 시 지원자에게 이메일 알림을 보내며, 감사 리포트(PDF/Excel)를 다운로드하는 "내부 기업용 대시보드" 웹 앱이다.
**참고 사항:** 핵심 AI 기능과 대시보드 검토 과정을 빠르게 확인하기 위해 로그인/인증 절차는 생략하고 관리자 권한을 가진 상태로 메인 화면에 즉시 진입하는 프로토타입이다.

**디자인 방향:**  
- 다크/라이트 모드 전환 지원 (기본: 라이트)  
- 색상 팔레트: Primary `#4F46E5` (인디고), Success `#10B981` (에메랄드), Danger `#EF4444` (레드), Warning `#F59E0B` (앰버), Muted `#6B7280`  
- 폰트: Pretendard (한글), Inter (영문)  
- shadcn/ui 스타일의 깔끔한 카드·테이블·다이얼로그 레이아웃  
- 상태별 뱃지: PASS(초록), FAIL(빨강), MANUAL_REVIEW(주황), PENDING(회색), APPROVED(파랑), REJECTED(빨강), SENT(초록)

## 화면 2 — 서류 업로드 영역 `/upload`

**레이아웃:** 대시보드 안의 서브 페이지 레이아웃 (사이드바 유지)

**Step 1: 지원자 및 서류 종류 선택**
- Select "지원자 선택": 김○○ (2026-0001), 이○○ (2026-0002) 등
- Select "서류 종류": 졸업증명서 / 자격증 / 경력증명서 / 성적증명서 등

**Step 2: 파일 업로드 영역**
- 드래그앤드롭 존 
  - "PDF, JPG, PNG 파일을 드래그하거나 클릭"
  - 중앙에 큰 구름/파일 아이콘

**Step 3: 업로드 시뮬레이션**
- "업로드 및 검증 시작" Button (Primary)
- 클릭 시 가짜 로딩 프로그레스바 또는 스피너 표시
- "AI 검증 중... (OCR 추출 중)" -> "검증 완료: PASS" 성공 메시지 모의 동작

## 화면 3 — 검증 결과 상세 모달 (Dialog)

**트리거:** 테이블의 "상세보기" 버튼 클릭  
**크기:** 최대 너비 800px, 스크롤 가능

**모달 헤더:**  
"검증 결과 상세 — 박○○ | 경력증명서" (오른쪽: `❌ FAIL` 뱃지)

**섹션 1: OCR 추출값 vs 기관 DB 비교**

| 항목 | OCR 추출 | 기관 DB 응답 | 일치 여부 |
|---|---|---|---|
| 성명 | 박재원 | 박재원 | ✅ 일치 |
| 발급번호 | CE-00123 | CE-00123 | ✅ 일치 |
| 발급일자 | 2024.03.15 | **2024.04.15** | ❌ **불일치** (빨간 하이라이트) |

**섹션 2: 분석 정보**
- AI 신뢰도: 91% (프로그레스 바)
- 불일치 사유: "⚠️ 발급일자 불일치: OCR 2024.03.15 ≠ 기관 DB 2024.04.15"

**하단 액션 (관리자 승인 처리 영역):**
- "✅ 수동 승인" Button (Success)
- "❌ 반려 처리" Button (Danger)
- "닫기" Button

## 화면 5 — 알림 관리 페이지 `/notifications`

**목적:** FAIL 판정 받은 지원자에게 서류 재제출 요청 이메일을 일괄 전송
**레이아웃:** 대시보드 서브 페이지

**상단 액션:**
- "선택 항목 발송 대상 1건" 
- "선택된 알림 이메일 발송" Button (Primary)

**알림 발송 대기 표:**
| 선택 | 지원자명 | 이메일 | 서류 종류 | 알림 상태 | 액션 |
|---|---|---|---|---|---|
| ☐ | 박○○ | p****@email.com | 경력증명서 | 📝 DRAFT | 발송 미리보기 |

**알림 미리보기 모달:**
- '미리보기' 클릭 시 이메일 템플릿(지원자명, 불일치 사유 자동 매핑됨)을 모달창으로 표시
- 텍스트 입력창에서 "추가 요청사항을 덧붙여입력" 가능
- 저장 후 [이 건 발송] 동작 지원
