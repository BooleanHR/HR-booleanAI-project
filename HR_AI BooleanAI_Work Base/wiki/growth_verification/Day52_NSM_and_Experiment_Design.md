# Day 52: 북극성 지표(NSM) 정의 및 실험 설계

## 📌 1. 북극성 지표 (North Star Metric)

HR BooleanAI의 핵심 가치는 **"채용 서류 진위확인 자동화로 만들어내는 신뢰된 채용 결정의 수"** 입니다.

| 구분 | 지표 |
| :--- | :--- |
| **North Star Metric (NSM)** | 월간 AI 자동 판정 완료 건수 (APPROVE + REJECT) |
| **보조 지표 1** | ESCALATE 비율 (낮을수록 AI 정확도 ↑) |
| **보조 지표 2** | 배치 완료까지 평균 소요 시간 (분) |
| **매출 지표** | 월간 검증 건당 과금 발생액 (Pay-per-scan) |

### NSM 선택 근거
- 진위확인 "완료" 건이 늘어날수록 담당자 업무 절감 시간이 비례 증가
- 담당자 업무 절감 → Retention 상승 → 매출 기반 강화의 선순환 구조

## 🧪 2. A/B 테스트 실험 설계

### 실험 1: AI 판독 결과 표시 방식 최적화 (가설 1 검증)

| 항목 | 내용 |
| :--- | :--- |
| **목표** | 담당자 의사결정 시간 80% 단축 (건당 평균 ≤ 30초) |
| **대조군 (A)** | 기존: AI 판정 결과만 텍스트로 표시 |
| **실험군 (B)** | 신규: 불일치 항목 하이라이트 + 자연어 근거 요약 카드 |
| **측정 지표** | `decision_confirm` 이벤트의 `decision_duration_ms` (p95) |
| **성공 기준** | B군의 p95 ≤ 30,000ms AND AI Override Rate ≤ 5% |
| **실험 기간** | 2주 (최소 50배치 완료 기준) |

### 실험 2: Self-Service 재제출 루프 (가설 2 검증)

| 항목 | 내용 |
| :--- | :--- |
| **목표** | 담당자 수동 개입 전화 90% 감소 |
| **대조군 (A)** | 기존: REJECT 시 담당자가 직접 지원자에게 연락 |
| **실험군 (B)** | 신규: Resend API 자동 이메일 + 72시간 유효 재제출 링크 |
| **측정 지표** | 재제출 링크 완료율, 회차별 담당자 수동 개입률 |
| **성공 기준** | 재제출 완료율 ≥ 85%, 수동 개입률 ≤ 10% |
