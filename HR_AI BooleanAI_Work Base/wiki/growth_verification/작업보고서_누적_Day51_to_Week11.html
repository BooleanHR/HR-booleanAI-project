📈 ARRR 퍼널 매핑 (HR BooleanAI)

B2B SaaS 기반 HR 서류 진위확인 솔루션의 사용자 여정을 5단계로 정의하고 각 단계에 실제 Pain 지표, 기준선, KPI를 연결합니다.

A
Acquisition (획득)
인사담당자 로그인 · KPI: WAU, 가입 전환율 · 채널: PoC 파트너 3곳 → 공공기관 931곳 확장
A
Activation (활성화) ⭐ 핵심 전환
첫 배치 등록 + 첫 1건 검증 성공 · KPI: 배치 등록 성공률 ≥ 95%, 첫 검증 시간 ≤ 10분
R
Retention (리텐션)
공채 회차별 반복 배치 사용 · KPI: 4주 리텐션 60% · 드라이버: 민원 전화 90% 감소 경험
R
Referral (추천)
감사 PDF + 결과 엑셀 유통 · KPI: 리포트 다운로드 수, ATS API 연동사 수
R
Revenue (매출)
건당 과금 또는 연간 구독 · KPI: LTV/CAC ≥ 3:1 · 기준선: 수기 비용 2,500원 → 목표 ≤ 1,250원

💡 핵심 비즈니스 가설 3종

가설 1 — AI 판독 추천 UI를 통한 의사결정 시간 단축
AI Reviewer Agent가 원본 서류 + RPA 캡처본을 3중 교차 검증(Triple Check Loop)하여 APPROVE/REJECT/ESCALATE 권고 + 한국어 근거 요약을 제공하면, 담당자 판정 시간이 12분 → 30초 이하 (96% 단축)될 것이다.
⏱ 기준선: 건당 12분 → 목표: p95 ≤ 30초
🤖 AI Override Rate ≤ 5%
가설 2 — Self-Service 소명/재제출 루프를 통한 민원 차단
REJECT 판정 시, 불일치 사유 + 72시간 유효 재제출 링크를 Resend API로 자동 발송하면, 담당자 수동 개입이 90% 이상 감소할 것이다.
📧 기준선: 민원 전화가 업무의 70~90% 차지 → Self-Service 처리율 ≥ 90%
🔗 재제출 완료율 ≥ 85% (72시간 이내)
가설 3 — 비개발자 PM의 기관 설정 hot-load 완결성
config/agency_config.json hot-load 엔진 + AI 기반 CSS 셀렉터 자동 추천 기능으로 비개발자 PM이 3분 이내 신규 사이트 설정~RPA 테스트 성공할 수 있을 것이다.
⚙️ 기준선: 개발자 의뢰 1~3일 → 목표: PM 직접 ≤ 3분
🤖 AI CSS 셀렉터 추천 성공률 ≥ 70%

⚠️ 수업 메모 코드 오류 교정 내역

수업 메모에 제공된 참고 코드에는 실제 Anthropic SDK와 호환되지 않는 파라미터들이 포함되어 있었습니다. 아래와 같이 교정하여 실제 동작하는 코드로 구현했습니다.

항목 메모 코드 (작동 안 함) 실제 구현 (교정 완료) 사유
모델명 claude-fable-5 claude-opus-4-5 존재하지 않는 미래/가상 모델명 → 실제 최신 Opus 모델 사용
Thinking 파라미터 thinking: { type: 'adaptive' } thinking: { type: 'enabled', budget_tokens: 10000 } 'adaptive' 타입은 SDK에 없음. 실제 Extended Thinking API 사용
비존재 파라미터 effort: 'max' 제거 OpenAI Responses API 파라미터. Anthropic SDK에 없음
비존재 파라미터 display: 'omitted' 제거 Anthropic SDK에 없는 파라미터
응답 파싱 agentResult.choices?.[0]?.message?.content response.content 배열 순회 (text/thinking 블록) OpenAI 형식. Anthropic SDK 실제 응답 형식은 content[] 배열
Prisma 필드 verificationStatus, reviewedAt, errorMessage status, rejectReason (실제 스키마 필드) 실제 schema.prisma에 없는 필드명 → 실제 필드로 교정

10주차 핵심 패러다임: Boundaries + Context 프롬프팅

마이크로 매니지먼트(단계별 지시) 대신 왜 필요한지 배경(Context)과 절대 넘으면 안 되는 경계(Boundaries)만 명시하고 에이전트의 자율성에 맡기는 방식입니다.

// 10주차 핵심: Boundaries + Context 프롬프트 구조 const systemPrompt = ` [CONTEXT] 당신은 채용 증빙 서류의 진위 여부를 최종 판정하는 'HR 자율형 리뷰어 워커'입니다. [BOUNDARIES - 제약 조건] 1. 최종 판정은 반드시 'APPROVE', 'REJECT', 'ESCALATE' 중 하나로만 도출 2. Jaro-Winkler 유사도 0.85 미만 → REJECT 2001년 이전 구버전 자격증 → 반드시 ESCALATE 3. 임의 데이터 생성 및 미지정 API 호출 금지 4. 내부 추론 과정 텍스트 노출 금지 — JSON 결과 포맷만 반환 [OUTPUT FORMAT] { "verdict": "APPROVE|REJECT|ESCALATE", "score": 0.0~1.0, "reason": "...", "discrepancies": [] } `;

실제 SDK 파라미터 (교정 완료)

const response = await client.messages.create({ model: 'claude-opus-4-5', // ✅ 실제 존재하는 모델 max_tokens: 16000, // ✅ thinking 사용 시 충분히 크게 temperature: 1, // ✅ thinking 활성화 시 temperature=1 필수 thinking: { type: 'enabled', // ✅ 실제 Extended Thinking API budget_tokens: 10000, // ✅ 사고 예산 (토큰) }, // ❌ effort: 'max' → 제거 (OpenAI 파라미터) // ❌ display: 'omitted' → 제거 (존재하지 않음) tools: [{ name: 'send_to_user_escalation', ... }], messages: [{ role: 'user', content: [이미지, 텍스트] }], });

ESCALATE 전용 도구 (Tool Use)

판정이 모호한 경우(특히 2001년 이전 구버전 자격증) Claude가 자율적으로 도구를 호출하여 담당자에게 수동 검토를 실시간 요청합니다.

도구명호출 시점전달 파라미터결과
send_to_user_escalation ESCALATE 판정 또는 판단 모호 시 reason (사유), excelExportRequired (엑셀 포함 여부) 대시보드 ESCALATE 배너 + 관리자 이메일 알림

Gemini 폴백 체계

Claude API Key 미설정, Refusal(거부) 발생, 또는 API 오류 시 기존 src/lib/ai/reviewer-agent.ts의 Gemini 시스템으로 자동 전환됩니다. 무중단 운영이 보장됩니다.

단계처리 내용조건결과
① Claude 검증 멀티모달(서류 이미지 + RPA 캡처본) 진위확인 실행 CLAUDE_API_KEY 미설정 시 Mock 모드 자동 전환 verdict / score / reason / discrepancies
② DB 동기화 VerificationJob (Claude 판정) + Applicant (status, rejectReason) + Batch (통계 카운터) 기존 verificationJobId 있으면 update, 없으면 create 3개 테이블 원자적 업데이트
③ REJECT 이메일 Resend API로 지원자에게 보완 서류 재제출 링크 자동 발송 verdict === 'REJECT' AND 지원자 email 존재 72시간 유효 고유 토큰 링크 포함 이메일
④ ESCALATE 알림 관리자(NOTIFICATION_EMAIL)에게 수동 검토 요청 이메일 발송 verdict === 'ESCALATE' 또는 toolCallTriggered 대시보드 링크 포함 관리자 알림

Mock 모드 (API Key 없이 즉시 테스트 가능)

✅ CLAUDE_API_KEY 미설정 시 Mock 모드 자동 전환
.env.local의 CLAUDE_API_KEY가 your-claude-api-key-here로 남아있으면 자동으로 Mock 결과를 반환합니다. 실제 키 없이도 전체 파이프라인 테스트가 가능합니다.
